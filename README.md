# Лабораторная работа

Дисциплина: "Разработка средств защиты информации"

Тема: "Рекурсивный шифратор/дешифратор"

## 1. Постановка задачи

Реализовать защиту данных пользовательских папок и файлов, находящихся в папке, а также подпапках, путем шифрования. Для доступа к данным исходной папки необходимо выполнить дешифрование.

Требования к реализации:

* интерфейс приложения - консоль;
* язык программирования - C++;
* основной класс, обеспечивающий шифрование и дешифрование, должен быть реализован как Singleton;
* входные данные: путь к папке или файлу для шифрования/дешифрования и пароль;
* необходим рекурсивный обход папок для обработки вложенных файлов.

## 2. Предлагаемое решение

Разработано консольное приложение `Encoder`, предназначенное для шифрования и дешифрования файлов и директорий.

Пользователь указывает путь к объекту файловой системы и пароль. Программа автоматически определяет, является ли указанный путь файлом или директорией, после чего выполняет соответствующую операцию.

При обработке директории выполняется рекурсивный обход всех вложенных подпапок и файлов.

### Зависимости проекта

В проекте используются следующие библиотеки и инструменты:

* Qt Core 5.x
* OpenSSL 3.x
* CMake 3.5 и выше
* Стандарт C++17

### Архитектура решения

Основные компоненты приложения:

#### main.cpp

Точка входа программы. Выполняет инициализацию Qt-приложения и запуск консольного интерфейса.

#### ConsoleInterface

Класс консольного интерфейса пользователя.

Функции класса:

* отображение меню;
* получение пути к файлу или директории;
* получение пароля пользователя;
* проверка корректности введённых данных;
* вызов операций шифрования и дешифрования.

#### CryptoManager

Основной класс системы, реализованный по шаблону Singleton.

Функции класса:

* шифрование файла;
* дешифрование файла;
* рекурсивное шифрование директории;
* рекурсивное дешифрование директории;
* проверка корректности входных данных;
* определение факта шифрования файла.

#### PasswordKeyDerivation

Класс для формирования криптографического ключа из пользовательского пароля.

Функции класса:

* проверка корректности пароля;
* получение 256-битного ключа с использованием PBKDF2-HMAC-SHA256.

#### crypto_constants.h

Файл с криптографическими константами:

* размер ключа AES-256;
* размер соли;
* размер nonce;
* размер тега аутентификации;
* количество итераций PBKDF2;
* сигнатура зашифрованного файла.

Рекурсивный обход директорий реализован средствами Qt с использованием класса `QDirIterator` и флага `QDirIterator::Subdirectories`.

### Алгоритм шифрования

Для защиты данных используется алгоритм AES-256-GCM.

Криптографический ключ формируется из пользовательского пароля с помощью функции PBKDF2-HMAC-SHA256.

Этапы шифрования:

1. Проверяется существование файла и возможность его обработки.
2. Выполняется проверка сигнатуры файла. Если файл уже был зашифрован программой, операция не выполняется.
3. Генерируется случайная соль размером 16 байт.
4. Из пароля и соли формируется 256-битный ключ при помощи PBKDF2-HMAC-SHA256.
5. Генерируется случайный nonce размером 12 байт.
6. Выполняется потоковое шифрование содержимого файла алгоритмом AES-256-GCM.
7. Формируется тег аутентификации размером 16 байт.
8. Результат сохраняется через механизм безопасной записи `QSaveFile`.
9. После успешного завершения операции исходный файл заменяется зашифрованной версией.

Структура зашифрованного файла:

| Поле    | Размер  | Описание |
|---------|---------|----------|
| MAGIC   | 7 байт  | Сигнатура `ENCFILE` — метка что файл зашифрован этой программой |
| VERSION | 1 байт  | Версия формата файла (текущая: `1`) |
| SALT    | 16 байт | Случайные байты для генерации ключа из пароля (PBKDF2) |
| NONCE   | 12 байт | Случайные байты для AES-GCM |
| Данные  | —       | Зашифрованное содержимое файла |
| TAG     | 16 байт | Тег аутентификации GCM — защита от подделки и повреждения |

### Алгоритм дешифрования

Этапы дешифрования:

1. Проверяется наличие сигнатуры `ENCFILE`.
2. Проверяется версия формата файла.
3. Из заголовка файла считываются соль и nonce.
4. Из введённого пользователем пароля повторно формируется ключ.
5. Выполняется дешифрование алгоритмом AES-256-GCM.
6. Проверяется тег аутентификации.
7. При успешной проверке восстанавливается исходное содержимое файла.
8. При неверном пароле или повреждении данных операция завершается ошибкой без изменения исходного файла.

### Безопасность

* SALT и NONCE генерируются случайно для каждого файла — одинаковые файлы с одинаковым паролем будут зашифрованы по-разному.
* Ключ уничтожается из памяти сразу после использования через `OPENSSL_cleanse`.
* Запись в файл происходит атомарно через `QSaveFile` — при ошибке исходный файл не повреждается.
* Неверный пароль обнаруживается через проверку тега аутентификации GCM.

## 3. Инструкция для пользователя

### Сборка проекта

```bash
mkdir build
cd build
cmake ..
cmake --build .
```

Перед сборкой необходимо установить:

* Qt Core;
* OpenSSL;
* компилятор с поддержкой стандарта C++17.

### Запуск

```bash
./Encoder
```

или в Windows:

```cmd
Encoder.exe
```

После запуска пользователю доступно меню:

```text
Выберите действие:
1. Зашифровать
2. Расшифровать
0. Выход
```

После выбора операции необходимо:

1. Ввести путь к файлу или директории.
2. Ввести пароль длиной от 8 до 256 символов.

### Пример проверки работы

1. Создать тестовую директорию с несколькими файлами.
2. Запустить программу.
3. Выбрать пункт «1. Зашифровать».
4. Указать путь к директории.
5. Ввести пароль.
6. Убедиться, что содержимое файлов стало недоступным для чтения.
7. Выбрать пункт «2. Расшифровать».
8. Указать тот же путь.
9. Ввести тот же пароль.
10. Убедиться, что исходные данные восстановлены.

### Ограничения

* пароль должен содержать от 8 до 256 символов;
* повторное шифрование уже зашифрованного файла не допускается;
* дешифрование возможно только для файлов, содержащих сигнатуру `ENCFILE`;
* при неверном пароле дешифрование завершается ошибкой;
* файлы изменяются на месте после успешного завершения операции;
* при обработке директории рекурсивно обрабатываются все вложенные файлы.
